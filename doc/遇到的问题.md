## 1.为什么没有触发 toolCalls呢？原因在哪儿

你当时那种“只输出计划、不触发 toolCalls”的原因，核心就是 LangChain4j 的工具调用不是强制的，它取决于模型在那次生成里是否“决定”调用工具；而你当前实现里有两个具体触发抑制点：

### 1) `langchain4j` 引擎是“一次性 chat”，没有 self 引擎那种循环逼它继续

`Langchain4jAgentEngine` 的主流程是：

- 构建 prompt（system + history + current user）
- `assistant().chat(...)` 只调用一次
- 如果模型这一次没主动发起 tool call，就直接结束（所以你看到 `rounds=1 toolCalls=0`）

它不像 `self` 引擎会“模型返回 tool_calls -> 执行 -> 再喂回去 -> 继续下一轮”。

### 2) 模型会倾向“先给计划”，但不一定会立刻调用工具

你给的问法是“比较…天气，并计算…”，模型很常见的行为是：

- 先输出 1-3 步计划（看起来很“规范”）
- 但不进入执行（尤其在它认为可以“凭常识回答”时）

这就是你看到的：只写了计划第一步就停。

### 3) 工具入参也会让模型更不愿意调用（之前）

`get_mock_weather` 之前只接受 `hangzhou/shanghai/...` 这种 key，模型如果传“杭州/上海”会失败，它在训练中对“工具调用失败”的代价很敏感，于是更容易选择“不调用”。

------

所以根因不在你前端/SSE，也不在 tool 本身“没注册”，而在于：
LangChain4j 的一次性生成 + 模型策略导致它没做工具调用。

这就是为什么我后来加了“确定性兜底”：只要输入明显包含天气/算式，就直接触发工具，绕过模型的“不调用”选择，从而保证 `toolCalls>0`。
