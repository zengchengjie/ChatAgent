## 项目简介

ChatAgent 是一个基于 AI 的智能对话应用，具有工具调用能力。它可以帮助您：

- 进行自然语言对话
- 使用工具执行特定任务（计算、天气查询、知识库搜索等）
- 管理多个对话会话
- 导出和分享对话内容

### 核心特性

- **智能对话**：基于大语言模型的自然语言理解
- **工具调用**：支持多种工具扩展功能
- **流式响应**：实时显示 AI 回复内容
- **会话管理**：创建、重命名、删除、导出会话
- **响应式设计**：支持桌面和移动设备
- **安全认证**：JWT 身份验证和会话隔离

## 快速开始

### 1. 启动应用

#### 使用 Docker（推荐）

```bash
# 克隆项目
git clone <repository-url>
cd ChatAgent

# 启动所有服务
docker-compose up -d
```

#### 手动启动

**后端：**
```bash
cd backend
./mvnw spring-boot:run
```

**前端：**
```bash
cd frontend
npm install
npm run dev
```

- 前端地址：http://localhost:5173
- 后端 API：http://localhost:8080/api

### 3. 登录系统

默认账号：

⚠️ **重要提示**：生产环境中请立即修改默认密码！

## 安装部署

### 系统要求

- **后端**：
  - Java 17 或更高版本
  - Maven 3.6+
  - PostgreSQL 14+ 或 SQLite（开发环境）
  - Redis 6+（可选，用于缓存）

- **前端**：
  - Node.js 16+
  - npm 或 yarn

- **其他**：
  - Docker 和 Docker Compose（可选）

### 环境配置

#### 后端配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/chatagent
    username: your_username
    password: your_password

dashscope:
  api-key: your_dashscope_api_key
  base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
  model: qwen-plus
  temperature: 0.7

agent:
  max-tool-calls-per-turn: 5
  max-tool-calls-total: 20
  tool-timeout-ms: 5000
```

#### 前端配置

编辑 `frontend/.env`：

```bash
VITE_API_BASE_URL=http://localhost:8080/api
```

### 数据库初始化

应用启动时会自动执行数据库迁移脚本（Flyway），位于 `backend/src/main/resources/db/migration/`。

### 生产部署

#### 使用 Docker Compose

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

#### 手动部署

**后端：**
```bash
# 构建 JAR 包
cd backend
./mvnw clean package -DskipTests

# 运行
java -jar target/chatagent-*.jar
```

**前端：**
```bash
# 构建
cd frontend
npm run build

# 部署到 Nginx
cp -r dist/* /var/www/chatagent/
```

## 使用教程

### 基础对话

1. **登录系统**
   - 打开应用登录页面
   - 输入用户名和密码
   - 点击"Sign in"按钮

2. **创建会话**
   - 点击左侧边栏的"+ New chat"按钮
   - 系统会自动创建一个新会话

3. **发送消息**
   - 在底部输入框中输入您的消息
   - 按 Enter 键或点击"Send"按钮
   - 等待 AI 回复

4. **查看回复**
   - AI 回复会实时显示在对话区域
   - 支持 Markdown 格式渲染

### 高级功能

#### 1. 使用工具

**计算器示例：**
```
用户：计算 123 * 456
AI：我来帮您计算 123 * 456。
[调用计算器工具]
AI：123 * 456 的结果是 56088。
```

**天气查询示例：**
```
用户：上海天气怎么样？
AI：我来查询上海的天气情况。
[调用天气工具]
AI：上海今天天气晴朗，温度 25°C。
```

**知识库搜索示例：**
```
用户：关于 Agent 开发有什么建议？
AI：让我搜索一下相关知识。
[调用知识库搜索工具]
AI：根据搜索结果，Agent 开发需要注意以下几点...
```

#### 2. 查看执行过程

右侧边栏会显示 Agent 的执行过程：

- **计划步骤**：显示 AI 的思考和计划
- **工具调用**：显示工具执行状态和结果
- **执行时间**：显示每个步骤的时间戳

#### 3. 响应式布局

- **桌面端**：三栏布局（会话列表、对话区域、工具面板）
- **平板端**：可折叠侧边栏
- **移动端**：全屏对话，侧边栏通过菜单按钮打开

## 工具使用

### 计算器工具

**功能**：执行数学运算

**支持运算符**：`+` `-` `*` `/` `(` `)`

**使用示例**：
```
计算 1 + 2
计算 (1 + 2) * 3
计算 100 / 4
```

**限制**：
- 仅支持数字和基本运算符
- 超时时间：500ms
- 结果范围：避免溢出和除零错误

### 天气查询工具

**功能**：获取城市天气信息

**使用示例**：
```
北京天气
上海今天天气怎么样
查询深圳的天气
```

**返回信息**：
- 天气状况（晴、阴、雨等）
- 温度
- 湿度（可选）

**注意**：当前为模拟数据，生产环境需接入真实天气 API。

### 知识库搜索工具

**功能**：搜索本地 Markdown 文档

**使用示例**：
```
搜索关于 Agent 开发的文档
查找 Langchain4j 的使用方法
```

**参数**：
- `query`：搜索关键词
- `k`：返回结果数量（默认 3）
- `minScore`：最小相似度分数（默认 0.0）
- `docTitleFilter`：文档标题过滤

**知识库配置**：

将 Markdown 文档放置在 `backend/src/main/resources/knowledge/` 目录，系统会自动索引。

## 会话管理

### 创建会话

1. 点击左侧边栏的"+ New chat"按钮
2. 系统自动创建新会话，标题为"New chat"
3. 开始对话

### 重命名会话

1. 在会话列表中找到要重命名的会话
2. 点击会话右侧的菜单按钮（三个点）
3. 选择"Rename"
4. 输入新标题
5. 按 Enter 键或点击保存按钮

### 删除会话

1. 在会话列表中找到要删除的会话
2. 点击会话右侧的菜单按钮
3. 选择"Delete"
4. 确认删除操作

⚠️ **警告**：删除操作不可恢复！

### 导出会话

1. 在会话列表中找到要导出的会话
2. 点击会话右侧的菜单按钮
3. 选择"Export"
4. 系统会下载 JSON 格式的会话文件

导出文件格式：
```json
{
  "session": {
    "id": "...",
    "title": "...",
    "createdAt": "...",
    "updatedAt": "..."
  },
  "messages": [
    {
      "id": 1,
      "role": "USER",
      "content": "...",
      "createdAt": "..."
    }
  ],
  "exportedAt": "..."
}
```

### 搜索会话

1. 在左侧边栏的搜索框中输入关键词
2. 系统会实时过滤会话列表
3. 点击匹配的会话查看内容

## 常见问题

### Q1: 忘记密码怎么办？

**A**: 当前版本使用默认账号 `admin/admin`。生产环境需要实现密码重置功能。

### Q2: 如何添加自定义工具？

**A**: 
1. 实现 `ToolExecutor` 接口
2. 添加 `@Component` 注解
3. 工具会自动注册到系统

示例：
```java
@Component
public class MyTool implements ToolExecutor {
    @Override
    public String name() {
        return "my_tool";
    }
    
    @Override
    public JsonNode toolDefinition() {
        // 返回工具定义
    }
    
    @Override
    public String execute(String args, String traceId) {
        // 执行工具逻辑
    }
}
```

### Q3: 如何配置不同的 LLM 模型？

**A**: 修改 `application.yml` 中的配置：

```yaml
dashscope:
  model: qwen-plus  # 或其他模型
  temperature: 0.7
```

### Q4: 流式响应中断怎么办？

**A**: 
1. 检查网络连接
2. 查看后端日志
3. 尝试重新发送消息
4. 如果持续出现问题，联系管理员

### Q5: 如何提高响应速度？

**A**:
1. 使用更快的 LLM 模型
2. 减少 temperature 值
3. 优化工具执行时间
4. 增加服务器资源

### Q6: 支持多用户吗？

**A**: 是的，系统支持多用户，每个用户只能访问自己的会话和消息。

### Q7: 数据存储在哪里？

**A**: 
- 会话和消息：数据库（PostgreSQL 或 SQLite）
- 知识库：文件系统 + 数据库
- 缓存：Redis（可选）

## 故障排除

### 问题 1：无法连接到服务器

**症状**：登录失败或 API 调用超时

**解决方案**：
1. 检查后端服务是否运行
2. 确认端口配置正确
3. 检查防火墙设置
4. 查看浏览器控制台错误信息

### 问题 2：工具调用失败

**症状**：AI 无法使用工具或工具执行超时

**解决方案**：
1. 检查工具配置
2. 查看后端日志
3. 确认工具依赖服务可用
4. 调整超时时间配置

### 问题 3：知识库搜索无结果

**症状**：搜索知识库时返回空结果

**解决方案**：
1. 确认知识库文档已正确放置
2. 检查文档格式（Markdown）
3. 验证嵌入模型配置
4. 尝试使用更通用的搜索词

### 问题 4：前端显示异常

**症状**：页面样式错乱或功能不可用

**解决方案**：
1. 清除浏览器缓存
2. 尝试使用其他浏览器
3. 检查浏览器控制台错误
4. 重新构建前端项目

### 问题 5：性能问题

**症状**：响应缓慢或系统卡顿

**解决方案**：
1. 检查服务器资源使用情况
2. 优化数据库查询
3. 增加缓存
4. 考虑使用 CDN
